ECS Fargate Microservice (Lab)

Sanitized demonstration. No company IP.

AWSECS (Fargate)DockerTerraformGitHub Actions

Goal

Run a simple containerized API on ECS Fargate with least-privilege IAM, provisioned by Terraform.

Architecture (Lab)

  • ALB (or public service) → ECS Service (Fargate) → Task (Docker image from ECR)
  • Logs → CloudWatch; config via env vars/SSM

Quick Start (local → ECR → ECS)

  1. Build app locally: 
    cd labs/ecs-fargate/app
docker build -t ecs-lab:demo .
  2. Create ECR repo (once) and push the image (replace REGION/ACCOUNT): 
    aws ecr create-repository --repository-name ecs-lab
aws ecr get-login-password --region REGION | docker login --username AWS --password-stdin ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com
docker tag ecs-lab:demo ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com/ecs-lab:demo
docker push ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com/ecs-lab:demo
  3. Use Terraform to wire ECS service (see repo folder). Then: 
    cd ../terraform
terraform init
terraform apply

What this demonstrates

  • Containerized API in Docker
  • Image lifecycle via ECR
  • IaC with Terraform (cluster/service/task)
